# Детектирование мобов в Minecraft

Проект по сравнению современных архитектур детектирования объектов (FCOS и YOLOv8) на специализированном наборе данных из игры Minecraft.

## Цель проекта
Обучить и сравнить две модели для автоматического поиска и классификации мобов (криперов, свиней, овец и др.) на кадрах из игры.

## Используемые архитектуры
1.  **FCOS (Fully Convolutional One-Stage)** — безъякорный (anchor-free) детектор, реализованный на базе фреймворка **MMDetection**.
2.  **YOLOv8s** — высокопроизводительная модель от **Ultralytics**, оптимизированная для работы в реальном времени.

## Набор данных
Используется датасет Minecraft, содержащий 17 классов мобов (bee, chicken, cow, creeper, enderman, pig, sheep, skeleton, spider, zombie и др.).
- Данные разделены на тренировочную, валидационную и тестовую выборки.
- Для FCOS используется формат COCO.
- Для YOLOv8 используется YAML-конфигурация с путями к изображениям и меткам.

## Сравнение моделей (на тестовом наборе)

| Модель | mAP (0.5:0.95) | mAP@50 | FPS (GTX 1080 Ti) |
| :--- | :---: | :---: | :---: |
| **YOLOv8s** | **0.563** | **0.809** | **~214** |
| **FCOS** | 0.219 | 0.388 | ~30 |

## Структура проекта
- `phase-1.py` - `phase-5-4.py`: Этапы подготовки данных, обучения и настройки моделей.
- `phase-6.py`: Скрипт для финальной оценки метрик на тестовом наборе.
- `phase-7.py`: Генерация визуальных примеров и финального PDF-отчёта.
- `artifacts/`: Результаты обучения, веса моделей, метрики и отчёты.
    - `report.pdf`: Подробный технический отчёт со сравнением моделей.
- `configs/`: Конфигурационные файлы для MMDetection.
- `datasets/`: Данные для обучения и тестирования.

## Как запустить оценку
1. Убедитесь, что установлены зависимости: `mmdet`, `mmengine`, `ultralytics`, `fpdf2`.
2. Запустите сбор метрик:
   ```bash
   python phase-6.py
   ```
3. Сгенерируйте визуальный отчёт:
   ```bash
   python phase-7.py
   ```
Результаты появятся в папке `artifacts/`.
